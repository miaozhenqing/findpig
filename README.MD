## 目录说明

- **cmd**：存放应用程序的入口文件。
    - `main.go`：程序启动文件，初始化应用、加载配置等。

- **config**：存放配置相关文件。
    - `config.go`：定义配置结构体和读取配置文件的方法。

- **internal**
    - **dao**：数据访问层，与数据库交互的代码。
        - `dao.go`：数据库操作方法。
    - **service**：业务逻辑层，处理业务逻辑。
        - `service.go`：业务逻辑实现。
    - **handler**：路由处理器，处理HTTP请求并调用服务层。
        - `handler.go`：HTTP请求处理函数。
    - **middleware**：中间件，如日志、认证等。
        - `middleware.go`：中间件实现。
    - **models**：数据模型，定义实体结构。
        - `models.go`：数据模型定义。

- **pkg**：公共工具包或第三方库的封装。
    - `utils.go`：通用工具函数。

- **routers**：路由配置。
    - `router.go`：定义API路由规则。

- **scripts**：脚本文件，如数据库迁移脚本等。
    - `migrate.sh`：数据库迁移脚本。

- **static**：静态资源文件（如果需要）。
    - (静态资源文件)

- **templates**：模板文件（如果需要）。
    - (模板文件)

- **vendor**：依赖包（可选，若使用模块化管理则不需要此目录）。
    - (依赖包)

- **go.mod** 和 **go.sum**：Go模块文件，用于管理依赖。
